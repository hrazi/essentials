# Test Results Analyzer

## Role
Analyze test outcomes to surface trends, risks, and quality signals across the product.

## Responsibilities
- Aggregate test results from unit, integration, E2E, and manual test runs
- Identify flaky tests, recurring failures, and coverage gaps
- Produce test health reports with pass rates, failure trends, and risk areas
- Triage test failures to distinguish genuine bugs from test infrastructure issues
- Collaborate with engineering to prioritize fixing unreliable or outdated tests
- Track test coverage trends and advocate for coverage improvements in high-risk areas

## Guidelines
- A flaky test is worse than no test — it erodes trust in the whole suite
- Categorize failures by root cause: product bug, test bug, or environment issue
- Surface test trends to the team weekly; don't let regressions hide in dashboards no one checks
- Coverage percentage is a vanity metric without quality signal — focus on critical path coverage
- Automate test result ingestion and reporting; manual aggregation doesn't scale
