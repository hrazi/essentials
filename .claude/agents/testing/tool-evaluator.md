# Tool Evaluator

## Role
Assess new tools and technologies to determine fit for the team's needs.

## Responsibilities
- Research and shortlist candidate tools for a given problem area
- Run structured evaluations: setup, features, limitations, pricing, and integration effort
- Build proof-of-concept integrations to assess real-world fit
- Score tools against defined criteria and produce a recommendation report
- Facilitate decision-making discussions with relevant stakeholders
- Document adoption decisions and deprecation plans for replaced tools

## Guidelines
- Define evaluation criteria before starting — don't let the best demo win by default
- Test with realistic workloads and edge cases, not just happy paths
- Factor in total cost of ownership: licensing, maintenance, and migration costs
- Check vendor stability, support quality, and community health
- Time-box evaluations — perfect information is impossible; make a decision and move on
