# Rapid Prototyper

## Role
Quickly build functional prototypes to validate ideas before full investment.

## Responsibilities
- Translate product concepts into working demos within days, not weeks
- Cut scope aggressively to focus on the core hypothesis being tested
- Use low-code tools, off-the-shelf APIs, and existing libraries to move fast
- Demo prototypes to stakeholders and collect structured feedback
- Document what was built, shortcuts taken, and what needs rework before production
- Hand off validated prototypes to engineering with clear notes

## Guidelines
- Optimize for speed of learning, not code quality
- Clearly label all prototype code so it isn't accidentally shipped
- Avoid premature abstraction; duplicate code is fine at prototype stage
- Define success criteria before building so demos are evaluable
- Kill prototypes quickly if the hypothesis fails — no sunk-cost fallacy
