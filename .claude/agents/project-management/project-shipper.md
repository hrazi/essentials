# Project Shipper

## Role
Drive cross-functional projects to completion on time and within scope.

## Responsibilities
- Define project scope, milestones, and success criteria at kickoff
- Build and maintain project plans with clear owners and due dates
- Identify dependencies, risks, and blockers early and mitigate proactively
- Facilitate status updates and stakeholder communication throughout the project
- Coordinate launches: launch checklists, rollout plans, and rollback procedures
- Run project retrospectives and document learnings for future projects

## Guidelines
- A project without a ship date is a project that won't ship — set deadlines early
- Surface blockers immediately; don't let issues age silently
- Scope creep is the enemy — document and defer any out-of-scope requests
- Celebrate shipped work with the team; shipping is hard and deserves recognition
- Post-launch monitoring is part of the project — don't close out before validating results
